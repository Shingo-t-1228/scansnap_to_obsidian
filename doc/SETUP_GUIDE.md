# 初心者向けセットアップガイド

このプロジェクト（`scansnap_to_obsidian`）を自分のPCで動かすための準備リストです。
プログラミングに慣れていない方でも、手順通りに進めるだけで環境が整うようになっています。

---

## なぜ `uv` を使うのか？

プログラミングの初学者が最初につまずくのは、プログラミングそのものよりも「環境構築」です。
このプロジェクトで `uv` を採用しているのには、以下の理由があります。

1. **PCの環境を汚さない**: プロジェクトごとに「隔離された小部屋（仮想環境）」を作るため、他のプログラムに影響を与えたり、逆に影響を受けたりしません。
2. **手順が劇的にシンプル**: 通常はいくつものコマンドが必要な手順が、`uv sync` という1つのコマンドだけで完了します。
3. **誰でも同じ結果になる**: `uv` がライブラリのバージョンを厳密に管理するため、「作成者のPCでは動くのに自分のPCでは動かない」というトラブルを防げます。

## 1. 必要なツールのインストール

まずは、プログラムを動かすための「土台」を整えます。

- [ ] **uv (ユーブイ)**: **【最重要】** Python本体の管理から、ライブラリの導入までをすべてこなす最新のツールです。
  - PowerShellで以下のコマンドを実行してインストールしてください。
    ```powershell
    powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
    ```
- [ ] **Git (ギット)**: コードをGitHubからダウンロードしたり管理したりするツールです。
  - [公式サイト](https://git-scm.com/downloads)からダウンロードしてインストールしてください。

> [!NOTE]
> `uv` をインストールすると、必要なバージョンの Python も自動的に用意されます。そのため、**個別に Python をインストールする手間は不要です**（すでにインストールされている場合も、それとは別に安全に動作します）。

## 2. プロジェクトのダウンロード

- [ ] **コードの取得**: 
  - 自分の好きなフォルダで PowerShell を開き、以下のコマンドを入力します。
    ```powershell
    git clone https://github.com/Shingo-t-1228/scansnap_to_obsidian.git
    cd scansnap_to_obsidian
    ```

## 3. 環境を自動で整える

- [ ] **ライブラリのインストール**:
  - フォルダの中で以下のコマンドを一回打つだけで、必要な準備がすべて完了します。
    ```powershell
    uv sync
    ```

## 4. AIモデル（Vision LLM）の準備

画像を解析するために AI の助けが必要です。

- [ ] **AIの選択（どちらか一方でOK）**:
  - **無料・ローカル（推奨）**: [LM Studio](https://lmstudio.ai/) をインストールし、`qwen2-vl` や `minicpm-v` などの Vision モデルをダウンロードして「Local Server」を起動します。
  - **有料・クラウド**: OpenAI (GPT-4o) などの API キーを用意します。

## 5. 設定ファイル（config.json）の作成

- [ ] **サンプルのコピー**:
  - `config/config.json.example` をコピーして `config/config.json` という名前に変更します。
- [ ] **内容の書き換え**:
  - `config.json` をメモ帳などで開き、以下の項目を自分の環境に合わせて書き換えます。
    - **Obsidianのフォルダパス**
    - **AIサーバーのURL** (LM Studioなら `http://localhost:1234/v1`)

## 6. 実行

- [ ] **プログラムの起動**:
  - 準備ができたら以下のコマンドで実行します！
    ```powershell
    uv run src/scansnap_to_obsidian.py
    ```
